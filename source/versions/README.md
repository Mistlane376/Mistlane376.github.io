# 📦 版本归档系统

## 概述

版本归档功能用于记录博客的所有版本更新历史和重要里程碑。

---

## 📂 文件结构

```
source/
└── versions/
    └── index.md  ← 版本归档页面
```

---

## 🎯 功能特性

### 1. 版本历史记录

- 记录每个版本的功能变更
- 包含新增、优化、修复项
- 详细的更新说明

### 2. 统计数据

- 版本对比（代码、文章、功能）
- 文件大小对比
- 更新频率分析

### 3. 里程碑标记

- 重要时间节点
- 重大更新标记
- 功能里程碑

### 4. 未来计划

- 已计划功能
- 远期目标
- 开发路线图

---

## 📝 版本命名规则

### 格式

```
主版本号.次版本号.修订号

例如：
v3.0.0 - 主版本3，次版本0，修订0
v2.1.0 - 主版本2，次版本1，修订0
```

### 主版本号（Major）

重大更新，可能包含不兼容的改动

**触发条件**：
- 完全重新设计
- 架构变更
- 功能模块重构

### 次版本号（Minor）

功能性更新，向下兼容

**触发条件**：
- 新增功能模块
- 重大优化
- 样式系统升级

### 修订号（Patch）

bug 修复和小改进

**触发条件**：
- Bug 修复
- 样式调整
- 文档更新
- 性能优化

## 🔄 更新流程

### 1. 更新版本归档

编辑 `source/versions/index.md`

在"版本历史"部分添加新版本：

```markdown
### v3.1.0 - 版本名称 (YYYY-MM-DD)

#### ✨ 新增功能

- 功能描述

#### 🐛 修复

- 修复描述
```

### 2. 更新统计数据

在"版本统计"表格中添加新版本数据

### 3. 更新里程碑

在"重要里程碑"部分添加新事件

### 4. 更新 CHANGELOG

编辑项目根目录的 `CHANGELOG.md`

### 5. 生成站点

```bash
npx hexo clean
npx hexo generate
npx hexo deploy
```
